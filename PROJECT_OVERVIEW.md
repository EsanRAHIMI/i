# نمای کلی معماری و اتصالات پروژه Ai Department

این سند بررسی جامع معماری فعلی، مسیرهای سرویس‌ها و اتصالات بین‌سرویسی برای اپلیکیشن Ai Department را ارائه می‌دهد.

## ۱. جزئیات اجرای محلی (Local Execution)

| سرویس | آدرس محلی (URL) | پورت | پوشه منبع |
| :--- | :--- | :--- | :--- |
| **Frontend** | [http://localhost:3000](http://localhost:3000) | 3000 | `/frontend` |
| **Backend API** | [http://localhost:8000](http://localhost:8000) | 8000 | `/backend` |
| **Auth Service** | [http://localhost:8001](http://localhost:8001) | 8001 | `/auth` |

## ۲. مسیرها و نقاط دسترسی API (Routes)

### 🔐 سرویس احراز هویت (Port 8001)
مدیریت چرخه عمر کاربر، امنیت و پروفایل را بر عهده دارد.
- `POST /v1/auth/register`: ثبت‌نام کاربر جدید.
- `POST /v1/auth/login`: ورود و صدور توکن JWT.
- `GET /v1/auth/google/authorize`: شروع فرآیند ورود با گوگل (OAuth).
- `POST /v1/auth/google/callback`: دریافت تاییدیه از گوگل.
- `GET /v1/auth/me`: دریافت اطلاعات پروفایل کاربر فعلی.
- `PATCH /v1/auth/me`: بروزرسانی اطلاعات پروفایل.
- `GET/PATCH /v1/auth/settings`: تنظیمات کاربر (واتس‌اپ، تقویم، حریم خصوصی).
- `POST /v1/auth/avatar/upload`: آپلود تصویر پروفایل.
- `GET /v1/auth/avatar/{filename}`: نمایش تصاویر آواتار.
- `GET /v1/docs`: مستندات Swagger سرویس.

### 🧠 سرویس بک‌اِند اصلی (Port 8000)
منطق تجاری و قابلیت‌های دستیار هوشمند.
- `WS /api/v1/voice/stream/{session_id}`: استریم صدای زنده (WebSocket).
- `/api/v1/calendar/*`: مدیریت تقویم گوگل (لیست کردن/ایجاد رویداد).
- `/api/v1/whatsapp/*`: مدیریت پیام‌های واتس‌اپ.
- `/api/v1/tasks/*`: مدیریت وظایف و یادآوری‌ها.
- `/api/v1/federated-learning/*`: ماژول‌های یادگیری غیرمتمرکز.
- `/api/v1/docs`: مستندات Swagger بک‌اِند.

### 🖥️ فرانت‌اِند (Port 3000)
مسیرهای اپلیکیشن Next.js.
- `/`: صفحه اصلی / لندینگ.
- `/auth/login`: صفحه ورود.
- `/auth/register`: صفحه ثبت‌نام.
- `/dashboard`: پنل اصلی کاربر.
- `/calendar`: رابط کاربری مدیریت تقویم.
- `/tasks`: رابط کاربری مدیریت کارها.
- `/settings`: مدیریت پروفایل و تنظیمات.

## ۳. اتصال سرویس‌ها و جریان داده (Connectivity)

### الگوهای ارتباطی
1. **Frontend -> Auth (8001)**: فراخوانی مستقیم برای تمام عملیات هویتی و تنظیمات پروفایل.
2. **Frontend -> Backend (8000)**: فراخوانی مستقیم برای قابلیت‌های برنامه، تعاملات هوش مصنوعی و صوت آنی.
3. **Internal Auth (JWT)**: هر دو سرویس از یک استراتژی RSA/JWT مشترک استفاده می‌کنند. بک‌اِند توکن‌ها را با کلید عمومی صادر شده توسط سرویس Auth تأیید می‌کند که احراز هویت بدون نیاز به شبکه (Stateless) را ممکن می‌سازد.
4. **Database Sharing**: سرویس‌ها از یک پایگاه داده PostgreSQL مشترک (`i_DB`) برای حفظ یکپارچگی داده‌ها استفاده می‌کنند.

### زیرساخت‌های مشترک
- **PostgreSQL**: پایگاه داده مرکزی برای ذخیره کاربران، وظایف و تنظیمات.
- **Redis**: محدودیت تعداد درخواست (Rate Limiting)، حافظه کش و هماهنگی وظایف پس‌زمینه.
- **MinIO/S3**: فضای ذخیره‌سازی برای دارایی‌های کاربر (آواتارها) و فایل‌های مدیا.

---

## ۴. وظایف هر بخش (Responsibilities)

- **Auth Service**: مدیریت هویت، چرخه عمر توکن، سیاست‌های امنیتی و پیکربندی کاربر.
- **Backend API**: مدیریت ویژگی‌ها، هماهنگی با هوش مصنوعی (OpenAI/ElevenLabs)، ادغام با ابزارهای اکوسیستم (Google/WhatsApp) و پردازش صوت.
- **Frontend**: تجربه کاربری (UI/UX) پاسخگو، مدیریت صوت زنده و ارائه یک داشبورد یکپارچه به کاربر.
